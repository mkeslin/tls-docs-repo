# Master records workshop

**Audience:** Any user who creates or links people, vehicles, property, or locations  
**Goal:** Search before add, open and edit a live master, understand snapshots, and know when to merge  
**Typical length:** ~90 minutes  
**Environment:** Training tenant  

## Prerequisites

- [New user orientation](new-user-orientation.md) completed (or equivalent)

## Agenda

| Block | Minutes | Guide | Exercise |
|-------|--------:|-------|----------|
| Overview | 10 | [Master records](../getting-started/master-records/README.md) | Open each rail icon |
| Search before add | 20 | [Search and add](../getting-started/master-records/search-and-add.md) | Find a person; attempt Add and use existing |
| Persons & vehicles | 20 | [Persons](../getting-started/master-records/persons.md) · [Vehicles](../getting-started/master-records/vehicles.md) | Edit a field; view Records / Associations |
| Property & locations | 15 | [Property](../getting-started/master-records/property.md) · [Locations](../getting-started/master-records/locations.md) | Property vs Evidence; Quick Locations if available |
| Snapshots | 15 | [Linking and snapshots](../getting-started/master-records/linking-and-snapshots.md) | Link a person from a sample citation or incident |
| Duplicates | 10 | [Duplicates and merge](../getting-started/master-records/duplicates-and-merge.md) | Discuss merge rights; demo if permitted |
| Wrap | 5 | — | Questions |

## Exit checklist

- [ ] Open master search from the right rail
- [ ] Demonstrate Use Existing instead of creating a duplicate
- [ ] Explain live master vs module snapshot
- [ ] Know who can merge duplicates
- [ ] Know where Master records docs live for self-serve
