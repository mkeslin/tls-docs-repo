# Role plan: Agency administrator

**Audience:** Agency admins / IT liaisons who maintain users, codes, and settings  
**Typical length:** Orientation + Admin workshop (+ module overviews as needed)

## Goals

- Open Admin and manage users with least privilege
- Maintain codes and offenses (ALL UPPERCASE descriptions)
- Know Modules tab is usually support-owned
- Point staff to the correct [role plan](README.md)

## Path

| Order | Curriculum / guide | Length |
|------:|--------------------|--------|
| 1 | [Shared foundation](README.md#shared-foundation-all-roles) | ~half day |
| 2 | [Admin workshop](../admin-workshop.md) | ~2 h |
| 3 | Skim [role plans](README.md) for every go-live audience | 30 min |
| 4 | Product overviews for modules you enable | as needed |

## Claims to verify

- Admin.User / Codes / Offenses / Agency / Officer as assigned
- Step-up works for Users admin

## Exit checklist

- [ ] Create/edit a training user and assign a minimal role
- [ ] Add or activate a non-locked code (UPPERCASE)
- [ ] Explain master merge is from Masters search, not customer Admin Master Merge
- [ ] Know how to read [FAQ](../../support/faq.md) / [Troubleshooting](../../support/troubleshooting.md) with end users
