# Incidents workshop

**Audience:** Officers, records staff, and supervisors who write or approve incident reports  
**Goal:** Create or open an incident, complete core tabs, understand workflow/supplements, and validate for IBRS  
**Typical length:** ~3 hours  
**Environment:** Training tenant; sample CAD call optional  

## Prerequisites

- [New user orientation](new-user-orientation.md) completed (or equivalent)
- Incidents (INC) access; Add claim if practicing manual create

## Agenda

| Block | Minutes | Guide | Exercise |
|-------|--------:|-------|----------|
| Overview | 10 | [Incidents overview](../rms/incidents/README.md) | INC vs Fire; CAD relationship |
| Search | 15 | [Search](../rms/incidents/search.md) | Incident Search + Arrest Search |
| Add / CAD | 20 | [Add](../rms/incidents/add.md) | Manual add **or** create from sample call |
| General | 20 | [General](../rms/incidents/general.md) | Synopsis, dates, location, associated calls |
| Narratives & offenses | 30 | [Narratives and offenses](../rms/incidents/narratives-and-offenses.md) | Write short narrative; add offense |
| Involved | 25 | [Involved persons](../rms/incidents/involved-persons.md) | Add victim + suspect (or arrestee) |
| Arrest / affidavit | 20 | [Arrests and affidavit](../rms/incidents/arrests-and-affidavit.md) | Arrest fields or affidavit print (as configured) |
| Property / evidence | 15 | [Property and evidence](../rms/incidents/property-and-evidence.md) | Add one property line if applicable |
| Workflow & validate | 25 | [Workflow](../rms/incidents/workflow-versions-and-approval.md) · [IBRS](../rms/incidents/print-attachments-history-ibrs.md) | Validate; discuss supplements |
| Related records | 10 | [Related records](../rms/incidents/related-records.md) | Associate citation or view group |
| Wrap | 5 | [Support](../support/README.md) | Questions |

## Exit checklist

- [ ] Open or create an incident and complete General + Narrative + Offense
- [ ] Add at least one involved person with a correct role
- [ ] Explain Draft vs approved / View-Only and when to use Versions
- [ ] Run Validate (or explain what it checks)
- [ ] Know where Incidents docs live for self-serve
