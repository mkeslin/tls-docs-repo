# Admin workshop

**Audience:** Agency administrators, IT liaisons, and supervisors who will maintain users and codes  
**Goal:** Open Admin safely, manage a user, maintain a code, and know what to escalate to Thin Line  
**Typical length:** ~2 hours  
**Environment:** Training tenant (prefer non-production)

## Prerequisites

- [New user orientation](new-user-orientation.md) completed
- At least one account with Admin claims in the training tenant
- [Master records workshop](master-records-workshop.md) recommended before discussing merge

## Agenda

| Block | Minutes | Guide | Exercise |
|-------|--------:|-------|----------|
| Open Admin | 10 | [Opening Admin](../admin/opening-admin.md) | Confirm menu items match claims |
| Users & officers | 35 | [Users, roles, and claims](../admin/users-roles-claims.md) · [Officers](../admin/officers.md) | Add or edit a training user; assign a minimal role; confirm login |
| Codes & offenses | 30 | [Codes](../admin/codes.md) · [Offenses](../admin/offenses.md) | Add or activate a safe test code; find an offense |
| Agency settings | 20 | [Agency settings](../admin/agency-settings.md) | Review Reports/Documents headers; note Modules as support-owned |
| Merge & boundaries | 15 | [Duplicates and merge](../getting-started/master-records/duplicates-and-merge.md) | Discuss Masters merge vs Thin Line–only tools |
| Audit / wrap | 10 | [Audit, security, and jobs](../admin/audit-security-jobs.md) | Optional Audit search; Q&A |

## Exit checklist

- [ ] Open Admin from the header and identify claim-gated menus
- [ ] Create or edit a user and assign permissions (training account)
- [ ] Add or edit a non-locked code with an ALL UPPERCASE description
- [ ] Know that module enablement changes go through Thin Line Support
- [ ] Know agency merge is from Masters search, not a customer Admin Master Merge screen
- [ ] Know where [Admin](../admin/README.md) docs live for self-serve
