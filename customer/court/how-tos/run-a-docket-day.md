# Run a docket day

![Court calendar](../images/court-calendar.png)

## Goal

Work today’s scheduled cases from the calendar: open each case, record the court outcome, and leave the next setting correct.

## Prerequisites

- Court Violations access
- Cases with appearance (or other hearing) dates for the session
- New cases already activated when they should be on the docket — see [Activate a new case](activate-a-new-case.md)

## Steps

1. Open **Court Violations** → **Calendar** (or start from the dashboard day agenda).
2. Focus on today’s date / the session you are calling.
3. Open the first case from the calendar entry.
4. Record what the court did, using the enabled action:
   - Defendant appeared → continue with plea / scheduling actions
   - Continuance only → **Modify appearance date** (or equivalent) to the new setting
   - Missed appearance → [Handle FTA and court warrant](handle-fta-and-court-warrant.md)
5. Return to the calendar and open the next case.
6. At the end of the session, spot-check that continued cases show the new date and FTA cases appear in the FTA queues.

## Expected result

- Outcomes are recorded on each called case.
- Continuances show the next date on the calendar.
- Missed appearances are on the FTA / show-cause path, not left as stale appearance dates.
