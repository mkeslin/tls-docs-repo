# Admin

Agency administration for Thin Line: users and permissions, officers, codes, offenses, and agency settings.

## What Admin is

**Admin** is the configuration area for people who maintain your Thin Line tenant. Day-to-day officers and clerks usually do **not** need Admin — they work in RMS, Court, Jail, or CAD modules.

| Area | Typical work |
|------|----------------|
| **Users, Roles, & Claims** | Create accounts, assign permissions, invite, terminate |
| **Officers** | Badge numbers, officer profiles used on citations and reports |
| **Codes** | Agency dropdown lists (CodeCommons) |
| **Offenses** | Offense / ordinance catalog |
| **Agency & Module Settings** | Agency identity, notifications, report/document headers |
| **Audit / Security / Scheduler** | Optional compliance and ops screens (when granted) |

## Who this guide is for

- Agency administrators and IT liaisons
- Records / court / jail supervisors who maintain codes and users
- Project leads preparing for go-live
