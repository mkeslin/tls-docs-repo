# Opening Admin

How to reach Admin and what controls what you see.

## Open Admin

1. Confirm you are in the correct **agency** (header name / logo).
2. In the top header, choose **Admin** (administrators only).
3. Thin Line opens Admin and lands on the first screen your claims allow (often Agency, Codes, Offenses, Users, or Officers).

If you do not see **Admin**, you do not have any admin claims — ask an existing administrator or Thin Line Support.

## Layout

| Area | Purpose |
|------|---------|
| **Left Admin drawer** | Claim-gated list of Admin screens |
| **Main content** | The selected Admin page |

You only see menu items your account can use. Two people at the same agency may see different Admin menus.

## Permissions (high level)

Admin access is **claim-based**. Common agency claims include:

| Claim area | Unlocks |
|------------|---------|
| Agency modify | Agency & Module Settings |
| Codes modify | Codes |
| Offenses modify | Offenses |
| User modify | Users, Roles, & Claims |
| Officer modify | Officers |
| Audits access | Audit History and Security Log |
| Scheduler admin | Scheduler & Jobs |
| CAD general admin | CAD Admin (when CAD is enabled for your agency) |

Your agency decides who holds these claims. Prefer a small set of trained administrators over broad Admin access.

## Step-up for Users

**Users, Roles, & Claims** may require an extra authentication step (**step-up**) before you can manage accounts. Complete the prompt when asked; without it, user administration stays locked for that session.

## Tips

- Finish Admin changes, then verify in the module (for example: new code appears in a citation dropdown; new user can sign in).
- Do not share temporary passwords in email threads that are not secured — treat invite credentials as sensitive.
- Module enablement and numbering under **Agency → Modules** are usually **view-only** for agencies — contact Thin Line Support to change them.
