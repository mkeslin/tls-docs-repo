# Codes

Maintain agency **code lists** (dropdown values) used across Thin Line.

## Open Codes

1. Open **Admin**.
2. Choose **Codes**.

Requires the codes-administration claim.

## How Codes works

1. Search or expand the **code type** tree (for example status lists, payment methods, or other agency types your tenant uses).
2. Select a type to see its codes.
3. **Add Code** when the type allows it, or open an existing code to edit.
4. Set the **description** and whether the code is **active**.
5. Save.

Inactive codes typically stop appearing for new work while historical records keep their stored values.

## Descriptions

Code descriptions shown in dropdowns should be **ALL UPPERCASE** (for example `NOT COMPLETED`, `SERVED`). Match your agency’s existing wording when migrating from a legacy system, but keep the uppercase convention for new values.

## Locked types and codes

Some lists are **system-locked** (often tied to state or federal reporting). You cannot add or edit those rows. If a required value is missing from a locked list, contact Thin Line Support — do not invent a parallel local code for the same meaning.

## Tips

- Prefer activating or editing an existing code over creating near-duplicates.
- After adding a code, open a module screen that uses that list and confirm the new option appears.
- Document who may approve new codes for Court, Jail, and RMS so lists stay consistent.
