# Working a close patrol

![Close Patrol](images/rms-close-patrol-search.png)

Day-to-day work on an open close-patrol detail.

## Tabs

| Tab | Purpose |
|-----|---------|
| **General** | Core request, location, dates, status, officers |
| **Notes** | Narrative notes for the request |
| **Emergency Contact** | Who to call if something is wrong at the location |
| **Logs** | Individual check / activity log entries |
| **Attachments** | Photos and files |
| **History** | Audit history (when you have RMS audit access) |

## Typical check workflow

1. Open the close patrol from Search.
2. Confirm **General** still matches the request (dates, status).
3. After a check, add a **Log** entry with date/time and outcome.
4. Add **Notes** or **Attachments** when something needs lasting context.
5. Update status when the request ends (per your agency codes).

## Tips

- Keep logs factual and timely — they are the operational history of the request.
- Use master **Locations** and **Persons** so the same address/contact stays consistent across modules.
- Close Patrol is not a substitute for a crime/incident report when one is required.
