# General

Header and case facts on the incident **General** tab.

## Typical General fields

| Area | Examples |
|------|----------|
| **Synopsis** | Short summary of the event |
| **Location** | Where it occurred (master location when used) |
| **Occurred start / end** | Event time window |
| **Reported** | When reported to the agency |
| **Incident call type** | Call / incident type code |
| **Primary officer** | Officer of record |
| **Follow-up** | Follow-up required / completed dates when used |
| **Internal note** | Agency-only note (permission-gated) |
| **Associated calls** | Linked CAD call sheets |
| **External call #** | Outside call number when applicable |

Complete required dates, location, and officer before seeking approval. Incomplete General data often fails IBRS validation later.

## Associated calls

Calls linked from CAD appear under associated calls on General. Open a linked call when you need dispatch context; keep the incident as the report of record for narratives and offenses.

## Tips

- Keep the synopsis short and factual; put the full story in **Narratives**.
- If Case Status or Exceptional Clearance is set from the header, understand your agency’s clearance codes before changing them — see [Workflow, versions, and approval](workflow-versions-and-approval.md).
- Confirm associated calls match the event you are documenting.
