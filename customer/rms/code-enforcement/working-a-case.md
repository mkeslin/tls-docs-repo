# Working a code enforcement case

## Tabs

| Tab | Purpose |
|-----|---------|
| **General** | Case identity, location, status, dates, officers |
| **Involved** | Persons linked to the case |
| **Narratives** | Written case narratives |
| **Offenses** | Ordinance / offense rows |
| **Timeline** | Chronological actions and follow-ups |
| **Attachments** | Photos, letters, files |
| **History** | Audit history (when permitted) |

## Typical workflow

1. Open the case from Search.
2. Confirm **General** status and location.
3. Maintain **Involved** parties from master persons.
4. Add **Offenses** that match the ordinance complaint.
5. Record inspections, notices, and outcomes on the **Timeline**.
6. Keep **Narratives** and **Attachments** aligned with what you did.

## Tips

- Timeline is the operational story supervisors rely on — update it when you act.
- Charts for code enforcement volume live under [Analytics](../../analytics/README.md) when you have access.
- When a case becomes a ticket or court matter, follow your agency path into [Citations](../citations/README.md) / [Court](../../court/README.md).
