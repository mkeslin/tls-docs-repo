# Property

![Masters — property](images/getting-started-modules-rail.png)

The shared **Property** master for items — distinct from Evidence room custody workflows.

## Open and edit

1. Right rail → **Property**.
2. Search by type, brand, model, serial #, description, and related fields.
3. Open the property card. Common tabs:

| Tab | Contents |
|-----|----------|
| **General** | Type, brand, model, serial #, color, description; drug-related fields when applicable |
| **Evidence** | Evidence numbers linked to this master property — see [Evidence](../../rms/evidence/README.md) |
| **Records** | Module records (for example incident property lines) |
| **Associations** | Linked persons and organizations |
| **Notes** / **Attachments** / **History** | Same pattern as other masters |

## Property master vs Evidence module

| Concept | Meaning |
|---------|---------|
| **Property master** | Shared identity of an item (what it is) |
| **Evidence** | That item taken into agency control with custody history |

Incident **Property** lines and evidence intake both hang off property identity. Search masters before creating duplicates so chain of custody does not split across two property files.

## Tips

- Serial numbers and unique marks are the best search keys.
- You generally **cannot merge** two property masters that share the same incident through evidence — see [Duplicates and merge](duplicates-and-merge.md).
- For booking into the property room, follow [Evidence — From incident property](../../rms/evidence/from-incident-property.md).
