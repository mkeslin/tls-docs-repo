# Locations

![Masters — locations](images/getting-started-modules-rail.png)

The shared **Locations** master for addresses and places.

## Open and edit

1. Right rail → **Locations**.
2. Search by address, city, type, and related fields.
3. Open the location card. Common tabs:

| Tab | Contents |
|-----|----------|
| **General** | Location type, address lines, city, state, ZIP (labels vary by type) |
| **Records** | Module records that used this location |
| **Associations** | Linked persons and organizations |
| **Notes** / **Attachments** / **History** | Same pattern as other masters |

## Quick Locations

**Quick Locations** (favorites-style menu) helps users jump to frequently used places when that feature is enabled for your role. Prefer Quick Locations for common reporting locations (station, jail, hospital) instead of retyping addresses.

## Tips

- Normalize addresses (street abbreviations, apartment numbers) so searches hit existing masters.
- Incidents, citations, and jail intake often store a **location snapshot** — update the live master when the canonical address changes, then fix module records if required — see [Linking and snapshots](linking-and-snapshots.md).
- Associate a business **Organization** when the location is a storefront.
