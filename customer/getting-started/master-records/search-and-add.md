# Search and add

Finding an existing master record before creating a new one.

## Open a master search

1. Click a master type on the right rail (for example **Persons**).
2. A slide-out opens titled for that type.
3. Enter search criteria (basic and advanced fields as shown).
4. Choose **Search** to refresh results.

Results appear as cards (or the current list UI). Open a result to view or edit the live master (when you have rights).

## Search before add

When you choose **Add**:

1. The system typically **re-searches** with your criteria.
2. If a likely match exists, you are prompted to **Use Existing** / **View Existing** instead of creating a duplicate.
3. If many matches appear, pick the correct existing record (or refine criteria).
4. Only choose **Add Record** / **Add as New** when you are sure no suitable master exists.

This habit is the single best way to keep people, vehicles, and property clean across Incidents, Citations, Warrants, Jail, and Court.

## Print results

**Print Results** (when available) prints the current search result set for offline review.

## Permissions

| Action | Needs |
|--------|--------|
| See the rail icon / search | Search rights for that master type |
| Add | Modify rights for that master type |
| Merge | Merge rights (separate from ordinary modify) |

If an icon is missing, ask your administrator — see [Support — FAQ](../../support/faq.md).

## Tips

- Enter enough identifying fields (name + DOB, plate + state, serial #) before Add.
- Do not create a new person because the first search was too vague — refine and search again.
- When linking from a module field (lookup mode), the same search-before-add rules apply.
