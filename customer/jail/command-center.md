# Command Center basics

Day-to-day use of the **Jail Command Center (JLM / JCC)**.

## Open the board

Open **Jail Command Center** from navigation. Select the correct **agency / facility** if prompted. The board is the live picture of people in custody for that facility.

## Main board areas

| Area | Purpose |
|------|---------|
| **Heartbeat / top bar** | Counts, search, Add Booking, Print Roster, Transfer Log, shortcuts to Intake |
| **Roster view** | List-oriented view of custody episodes |
| **Layout view** | Location / cell oriented view of who is where |
| **In Booking Process** | Draft, in-process, and ready-for-acceptance bookings not yet accepted |
| **Location rail** | Destinations for moves (holding, intake, gym, court, offsite, and configured cells) |
| **Episode drawer** | Detail for the selected person in custody |
| **Work queue** | Due tasks such as cell checks and medication |
| **Live feed** | Recent activity / presence style updates when opened |

## Roster vs Layout

Toggle between **Roster** and **Layout** depending on how you run the floor:

- **Roster** — scan who is in custody and open an episode quickly
- **Layout** — see occupancy by location / cell for housing decisions

Selection of a person should open the episode drawer with matching custody context.

## Filters

Common filters include:

- Suicide watch / critical attention
- Unassigned housing
- Onsite vs offsite
- Hide empty locations

Clear filters when you need the full board again.

## In Booking Process

Use this list to:

- See bookings still in intake
- **Open booking** to finish steps or accept
- Spot **ready for acceptance** work that is waiting on a supervisor or intake officer

## Live updates

Moves, new bookings, and task activity from other users should refresh the board without a full reload. If the connection drops, reconnect or refresh and confirm the roster before releasing or moving someone.
