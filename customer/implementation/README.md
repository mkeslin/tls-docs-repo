# Implementation

Working materials Thin Line and your agency use together while moving onto the Thin Line Platform.

These pages are **customer-safe**. They do not include Thin Line internal engineering procedures. Fill templates with agreed facts only — prefer “TBD” or “Pending assessment” over invented dates.

## Documents

| Document | Purpose |
|----------|---------|
| [Discovery brief template](discovery-brief-template.md) | Agency-specific pre-read pattern (copy into the engagement workspace; do not send blank) |
| [Implementation kickoff agenda](implementation-kickoff-agenda.md) | 60–90 minute kickoff agenda with notes and action register |
| [Preliminary implementation plan](preliminary-implementation-plan.md) | Working plan after discovery; before all assessments are complete |
| [Implementation action register](implementation-action-register.md) | Living list of actions, decisions, risks, and gaps |
| [Customer implementation workbook](customer-implementation-workbook.md) | Modular framework: what to collect, when, and who |
| [Implementation working session workbook](implementation-working-session-workbook.md) | Fill-in template for a live working session (reusable) |
| [Configuration discovery workbook](configuration-discovery-workbook.md) | Approve product configuration decisions (modular by purchased products) |
| [Go-live and hypercare brief](go-live-and-hypercare-brief.md) | Customer-facing notice of cutover and the hypercare window |

## How to use this pack

1. Before discovery, send a short **agency-specific discovery brief** (copy the template into the engagement workspace and customize).
2. Start the first joint meeting with the **kickoff agenda** and/or **working session workbook**.
3. Capture open work in the **action register**; keep the **preliminary plan** current as discovery lands.
4. Use the **workbooks** in working sessions (configuration, collection, live notes).
5. Issue the **go-live and hypercare brief** when the cutover window is confirmed — not before.
6. Point administrators at [Admin](../admin/README.md) and trainers at [Training](../training/README.md) as products approach readiness.
