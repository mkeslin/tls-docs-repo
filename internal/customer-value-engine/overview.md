# Customer Value Engine overview

Thin Line’s Customer Value Engine is how the company acquires customers, delivers the platform, supports ongoing use, expands relationships, and turns success into advocacy.

## Customer value stream

See [Customer value stream](customer-value-stream.md) for the five phases and full stage list.

| Phase | Outcome |
|-------|---------|
| **Acquire** | Signed customer ready to implement |
| **Deliver** | Customer live on Thin Line Platform |
| **Operate** | Reliable support and product updates |
| **Expand** | Deeper adoption and additional modules/sites |
| **Advocate** | References, referrals, and community |

## Maturity scale

Each stage is scored 1–5:

| Score | Meaning |
|------:|---------|
| 1 | Founder-driven |
| 2 | Documented |
| 3 | Standardized |
| 4 | Automated |
| 5 | Scalable |

## Improvement framework

Every process should be evaluated in this order:

1. **Simplify** — remove unnecessary effort or complexity.
2. **Standardize** — define one repeatable way to perform the work.
3. **Automate** — use software to remove repetitive work.
4. **Delegate** — transfer ownership with clear expectations.
5. **Scale** — design the process to handle substantially more volume.

Stage pages capture current owner, tooling, capacity, cycle time, maturity, what would break first, and 10x ideas across those five lenses.
