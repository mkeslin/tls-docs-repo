# Customer value stream

The Customer Value Engine is organized into five phases. Each stage page documents desired outcome, current process, inputs/outputs, ownership, tooling, capacity, maturity, and improvement opportunities.

## Acquire

Awareness → Cold leads → Warm leads → Hot leads → Demo → Proposal → Contract → Invoice

| Stage | Desired outcome | Owner | Maturity |
|-------|-----------------|-------|----------|
| [Awareness](acquire/awareness.md) | Generate interest | **Decision needed** | 1 |
| [Cold leads](acquire/cold-leads.md) | Identify prospects | Fugate | 2 |
| [Warm leads](acquire/warm-leads.md) | Qualify interest | Fugate | 2 |
| [Hot leads](acquire/hot-leads.md) | Confirm interest | Fugate | 2 |
| [Demo](acquire/demo.md) | Demonstrate value | Fugate | 1 |
| [Proposal](acquire/proposal.md) | Present solution | Fugate | 3 |
| [Contract](acquire/contract.md) | Execute agreement | Fugate | 3 |
| [Invoice](acquire/invoice.md) | Begin financial relationship | Keslin | 4 |

- [Acquire overview](acquire/README.md)
