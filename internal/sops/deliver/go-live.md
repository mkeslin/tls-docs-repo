# Go live

**Lifecycle step:** 7 · Go live  
**Milestone outcome:** Customer live in production  
**Document type:** Phase overview  
**Status:** v1  

Engagement status: ☐ Not started · ☐ In progress · ☐ Complete

Assessment: [Go-live readiness assessment](../../assessments/go-live-readiness-assessment.md)  
CVE stage: [Go live](../../customer-value-engine/deliver/go-live.md)

---

## Purpose

Cut over to Thin Line as the system of record for agreed modules—customer is **live** in production—under an authorized Go / Conditional Go decision, then open hypercare.

---

## Inputs

| Input | Source |
|-------|--------|
| Prior lifecycle phases Complete or Not applicable | Implementation workspace board |
| Completed readiness assessment | [Go-live readiness assessment](../../assessments/go-live-readiness-assessment.md) |
| Formal decision | [Go-live decision record](../../templates/go-live-decision-record.md) |
| Customer notice | [Go-live and hypercare brief](../../../customer/implementation/go-live-and-hypercare-brief.md) |
| Cutover checklist | [Go-live runbook](go-live-runbook.md) (engagement copy) |

---

## Activities

1. Complete the [Go-live readiness assessment](../../assessments/go-live-readiness-assessment.md) (evidence from prior phases).  
2. Record **Thin Line recommendation** and **customer authorization** on the [decision record](../../templates/go-live-decision-record.md).  
3. Issue or update the [customer go-live and hypercare brief](../../../customer/implementation/go-live-and-hypercare-brief.md).  
4. Execute the [Go-live runbook](go-live-runbook.md) (freeze/sync if applicable, verify, cutover, smoke, triage).  
5. Open the hypercare window with named coverage; hand open issues into Phase 8 tracking.  

Specialist work (infrastructure, migration, configuration, integrations, training) may be done by the **Implementation lead** unless a separate hat is assigned—see [Roles](implementation-roles-and-responsibilities.md).

---

## Outputs

| Output | Notes |
|--------|--------|
| Decision record | Go / Conditional Go / No Go |
| Customer brief issued | Shared with agency |
| Production use for agreed modules | Or explicit stop / rollback recorded |
| Hypercare start | Duration and contacts named |
| Updated workspace / action register | Cutover issues tracked |

---

## Exit criteria

- [ ] Readiness assessment Complete (or Conditional Go conditions satisfied and recorded)  
- [ ] Thin Line recommendation **and** customer authorization recorded  
- [ ] Customer using Thin Line for agreed production workflows **or** No Go / rollback explicitly recorded  
- [ ] Hypercare ownership and window defined (when Go proceeded)  
- [ ] Lifecycle board Phase 7 updated  

---

## Document maturity vs engagement status

This page’s **Status: v1** is document maturity. Engagement work uses: Not started · Waiting on customer · Waiting on Thin Line · In progress · Blocked · Ready for validation · Complete · Deferred · Not applicable. See [Workspace standard](implementation-workspace-standard.md).
