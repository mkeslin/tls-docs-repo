# Working session workbook template

**Document type:** Template  
**Status:** v1  
**Audience:** Thin Line only (Implementation · Sales on discovery)  
**Purpose:** Meeting notebook for discovery / working sessions—organized around implementation, not free-form notes  

**Not for the customer.** The customer receives the [Discovery brief](../../customer/implementation/discovery-brief-template.md) before the meeting. This workbook stays in the **engagement workspace**.

---

## How to use

1. Copy this file into the engagement workspace.  
2. Rename it for the agency (e.g. `Working-Session-Workbook-<Agency>.md`).  
3. Keep, drop, or add section headings to match the discussion you want (mirror the discovery brief objectives).  
4. During the meeting, capture under each heading—do not turn this into a customer homework packet.  
5. Immediately after, move durable actions into the [Action register](../../customer/implementation/implementation-action-register.md).  
6. After one or two real engagements, promote useful heading changes back into this GitBook template.

### Document flow

```text
Before     Discovery brief (customer)
During     Working session workbook (Thin Line)  ← this document
After      Action register
Next       Migration assessment (if needed)
Then       Preliminary implementation plan
Then       Formal kickoff (when contract / path is clear)
```

Each artifact has one job. Do not expand this workbook into a second discovery brief or a configuration workbook.
