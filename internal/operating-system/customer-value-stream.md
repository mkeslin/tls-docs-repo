# Customer value stream

The operating model is organized into five phases. Each stage page documents purpose, trigger, inputs, outputs, ownership, tooling, and improvement opportunities.

## Acquire

Awareness → Cold leads → Warm leads → Hot leads → Demo → Proposal → Contract → Invoice

- [Acquire overview](acquire/README.md)

## Deliver

Data conversion → Bootstrap environment → Training → Go-live / configuration

- [Deliver overview](deliver/README.md)

## Operate

Support → Product updates

- [Operate overview](operate/README.md)

## Expand

Customer success → Expansion

- [Expand overview](expand/README.md)

## Advocate

Reference → Referral → Community

- [Advocate overview](advocate/README.md)

## Stage page standard fields

Every stage page should ultimately document:

- Purpose
- Trigger
- Inputs
- Outputs
- Owner
- Current process
- Tooling
- Capacity
- Cycle time
- KPIs
- Failure conditions
- What would break first at higher volume
- Simplify / Standardize / Automate / Delegate / Scale opportunities
- Product responsibilities
- Related SOPs, checklists, and templates

Until filled from real process interviews, fields use `TODO:` markers.
