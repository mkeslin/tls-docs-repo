# Home

Staff-only operating documentation for Thin Line Software.

This space is the durable source of truth for how Thin Line runs the business — not a general wiki, and not customer-facing Help. Use it to find strategy, the Customer Value Engine, procedures, and playbooks that should stay consistent as the company grows.

## Why this exists

Thin Line is building a unified public safety platform (Thin Line Platform) for small and mid-sized Texas agencies. Scaling that work depends less on heroics and more on shared standards: one way to sell, implement, support, and expand.

These pages capture those standards so new work compounds instead of reinventing process each time.

## Document types

| Type | Meaning |
|------|---------|
| Policy | A governing rule |
| SOP | A repeatable procedure |
| Guide | Explanatory instructions with judgment involved |
| Checklist | Verification steps |
| Template | Reusable starting material |
| Reference | Factual supporting information |

When something is unknown or undecided, pages use `<mark style="color:$danger;">**TODO:**</mark>` or `<mark style="color:$danger;">**Decision needed:**</mark>` markers instead of inventing answers.
