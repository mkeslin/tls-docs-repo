## General Changes

<!-- - Added a new Criminal Trespass module (initial release, behind an agency feature flag) -->
<!-- - Added automatic record-number generation for the Criminal Trespass module -->
- Added CJIS IA-11 session management compliance enhancements
- Added a security banner for active sessions, with explicit handling for unauthenticated users
- Hardened security service initialization, error handling, and authentication checks
- Added a backlog item to harden Descope cookie revocation on logout (BL-020)
- Made other small improvements and fixes

## Jail Intake

### Reports

- Added a Booking Medical Report (PDF) with comprehensive medical history, allergies, and medications
- Added a Booking Mental Health Report (PDF) generated from mental health screening
- Added a Booking Classification Notice Report (PDF), including the classification flowchart
- Added a Booking Custody Report and a Booking Property Report with consistent layouts
- Added a Booking Detail Report
- Added PDF generation for jail intake step reports, with completion validation before generation
- Added person details with a normalized charge line presentation
- Refactored report generation to remove legacy HTML/Dink output and standardize on QuestPDF
- Improved layout consistency, headers, footers, and section titles across reports

### Functionality

- Added soft-delete for bookings and related entities, with cancellation token support
- Added an authorization check on the release booking action
- Added the ability to delete a booking from the booking details view
- Added validation enhancements to the custody step
- Improved name handling on jail intake entry and booking step components
- Reorganized jail intake search options and results for clarity
- Renamed "Medical Log" to "Medication Log" across components and routes
- Added PDF view and download in the Medication Log and Observation Pass searches

### UI / Layout

- Refactored the booking details header with the TlsAutocomplete component and a `preserveCurrentStep` parameter
- Refactored the custody, medical, mental health, and classification step components for improved layout and styling
- Added a "final locked view" to the classification step
- Removed the housing assignment step components pending further work
- Disabled the DOCS step pending further work

## Jail Management (JLM)

- Added an Offsite Transfer Wizard with a refined checklist and the ability to abandon a draft custody transfer
- Added a Release Booking dialog with an improved layout and identity confirmation
- Refactored the JLM episode drawer to toggle button actions based on booking ID
- Added roster PDF output to the board container
- Added observation pass milestones and a sliding ROUND-cadence implementation
- Added missed-round pass handling and an improved scheduler that uses a periodic timer
- Added medication deactivation and task status updates when a custody episode is released
- Refactored the JlmService and related components to remove the inter-agency transfer requirement
- Refactored the observation pass controllers and integrated them with the JLM module
- Added silent loading and polling improvements to the JLM board container
- Added drag-and-drop UI responsiveness improvements
- Added a Primary Security Designation Strip with unit tests
- Removed the open-person feature from JLM cards in favor of streamlined navigation

## Court Violations

- Added a SQL script for a court violation collections query
- Refactored the post-judgment violation backfill and added warrant synchronization queries
- Added compliance phase tracking to the court dashboard
- Refactored the court violation health view layout

## Document Vault

- Enhanced the document vault with global file handling, directory existence checks, and improved unit tests
- Updated document vault routes and the UI for managing booking-related files
- Replaced the PDF icon with a more reliable alternative

## Architecture Changes

- Added Swashbuckle annotations to enhance API documentation
- Added integration tests for SQL Server, including HTTP tests for incidents and warrants
- Added unit tests across the API surface (image handling, claim handlers, observation pass concurrency, soft-delete, ObservationPassController, JlmObservationController)
- Added an `IncodeCourtMigrationTool` project for converting from Incode
- Added a `CrimeStar` migration tool with a `--drop-empty` option to remove empty staging tables
- Added a sandbox sample-ID service and controller to support migration validation
- Added a TlsSelect component, integrated into the sandbox controls
- Added a new API/UI control-flow brace style rule (single-line bodies must use braces)
- Refactored navigation drawer components to use dynamic component loading
- Updated content security policy to allow GitHub raw content
- Added active-user filter and middleware safety for null IP addresses
- Updated NuGet authenticate task version in pipeline YAML
- Updated client-side libraries to the latest versions
- Updated server-side libraries
- Made other small improvements and fixes
