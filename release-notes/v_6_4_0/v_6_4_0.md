# Version 6.4.0

Welcome to the 6.4.0 release of Thin Line RMS/CAD. There are a number of updates in this version that we hope you like.

## Offenses & Court Violations

- Updated the window tint offense description so officers can find it by searching for "tint"
- Improved court violation menu options based on agency online payment and collections settings

## General Changes

- Made other small improvements and fixes

# Version 6.4.1

## Jail Management (JLM)

- Improved observation pass scheduling for automated and manual cell checks
- Added ability to cancel an observation pass, with schedules adjusted correctly
- Historical cell-check views now include released inmates
- Added quick-select buttons for common cell-check observations
- Improved cell-check notes, row selection, and keyboard shortcuts in the Pass Runner
- Added notification sounds and in-app audio settings for jail tasks
- Added real-time "booking in progress" notifications on the JLM board
- Improved task alert sounds in the jail work queue

## Jail Intake

- Improved offense code and level display on booking charges and reports
- Improved arresting officer name display on booking reports
- Improved date and time handling on jail intake reports
- Improved booking charge formatting on reports

## Incidents

- Added a Juvenile incident tag
- Made incident tags editable (except Cargo Theft)
- Improved how incident tags display on incidents
- Locked selected incident tags used for reporting control

## Accounting & Collections

- Improved date range selection on collections and bond reports and batch screens
- Standardized accounting fee descriptions
- Added an Animal Control fee

## Administration & Mobile Citation

- Added a mobile citation print preview on agency admin
- Improved help text and layout on agency admin general settings

## General Changes

- Made other small improvements and fixes

# Version 6.4.2

## Court Violations

- Added Violate Promise to Appear (VPTA) filing for court violations (agency-configurable)
- Improved payment plan handling so payment plans are managed through accounting instead of disposition-plan conditions
- Improved payment allocation when credits reduce what is still owed
- Users can enter a citation number or court violation number when verifying payments
- Clearer payment error messages for declined and failed card payments
- Added warrant supplement details (financial and defendant identification) on court violation warrant reports
- Added follow-up date filtering on court violation search (past due and upcoming)
- Removed empty payment plan shells from disposition program management

## Collections

- Added a dedicated Collections module with role-based access for court and collections workflows
- Improved collections referral search, sorting, and date filtering
- Improved collections remittance, disbursement, and activity report search
- Added collections referred accounts reporting with PDF and CSV export
- Added CSV export for collections activity and remittance reports
- Added collections disbursement batch detail reporting
- Added general ledger export options for collections disbursement batches
- Added support for supplemental item-level collections referrals when new fees are added on referred cases
- Improved collections referral satisfaction when a referred balance is paid in full
- Improved vendor name display and date handling on collections disbursement screens

## Administration

- Improved agency admin layout for court and collections related settings

## General Changes

- Made other small improvements and fixes
