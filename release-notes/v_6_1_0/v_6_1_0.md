## General Changes

- Updated terminology in court workflow ("Deferred disposition" → "Court program"; "Schedule Show Cause" → "Mark Failed to Comply"; "Add Diversion" → "Grant Deferred Disposition")
- Added pin/unpin favorites and disposition plan and payment plan indicators on court violation results
- Added violator-name display ("last first" format) and community-service-hours calculation
- Added a sync debug feature for full-support users
- Made other small improvements and fixes

## Court Violations

### State Machine & Workflow

- Moved the court violation state machine to the API for consistency across clients
- Added new state transitions: Change Plea, Reject/Withdraw Plea, DismissCase (PreTrial), idempotent MarkFailureToAppear, MarkComplianceOrderSuccessful, and Vacate Judgment (with required event note)
- Added pre-plea handling, transfer, and void support
- Added age-at-offense calculation and juvenile gating
- Removed deprecated procedural states

### Work Queues

- Added a Court Violation Work Queue system, with stable codes and queue counts
- Added queues for Pending Payments, Missed Installments, Payment Plan Time Fee, Deferred Disposition Ready to Close, Follow-up, Show Cause, and Bond
- Added batch operations: batch mark active, batch set show cause date, and batch removal
- Renamed "Court Proceedings" to "Work Queues" in the navigation drawer

### Payment Plans

- Added a payment plan dialog with the ability to view active and historical plans
- Added the ability to cancel a payment plan and updated payment-plan handling for dismissed violations
- Added a Payment Plan Fee Queue page for fee assessment

### Bonds & Disposition Plans

- Added Court Violation Bond support, including bond status and active-bond retrieval
- Added disposition plans and conditions, with completion eligibility and a redesigned dialog
- Added attorney tracking on disposition plans
- Replaced explicit "expected end on" with derivation from active conditions

### Compliance & Show Cause

- Added "Set Show Cause Date" functionality and button
- Added the show-cause hearing date to court violation reports
- Added compliance due-date detection, extension, and agency time-zone handling
- Added late notice letter with online-payment instructions
- Refactored "Mark Failed to Comply" handling for missed payments and compliance dates

### Reports & Documents

- Added a Court Violation Overview report
- Updated deferred disposition, installment agreement order, and show cause documents
- Added an Avery 8160 label sheet for FTA initial appearance addresses
- Added a Pre-Trial Setting Notice document
- Improved the installment agreement order with cause-number handling and streamlined display

### Health & Validation

- Added new health-issue types: PAID_IN_FULL_NOT_CLOSED, JUDGMENT_BEFORE_PLEA, PAYMENT_PLAN, PAYMENT_PLAN_ZERO_BALANCE, DISPOSED_WITH_ACTIVE_PLANS

## Accounting

- Added a Simple Accounting Dashboard that replaces the previous accounting dashboard
- Added deposit batches with posting validation, total recalculation, and processor-payment removal
- Added settlement batches with court total amounts and a payment-settlement preview
- Added payout batches with creation, total calculation, net payout, and webhook reconciliation
- Added card payout view (no date filter)
- Added transaction lines (journal entry lines) and refactored batch posting and revenue allocation
- Switched accounts from internal IDs to account codes
- Added soft-delete support to accounts
- Added a State Processing Fees account
- Added new reports: Payment Settlement Report, Court Violation Statement
- Added time-credit support on payment receipts
- Added an accounting manual

## Jail Intake

- Stubbed in a new Jail Intake / Booking module with a guided wizard
- Added booking steps for custody, property, charges, medical assessment, classification, housing assignment, and a modular assessment framework
- Added booking search with arrest-agency filtering
- Added a SignalR-based real-time pipeline for jail intake
- Added step assignee management, locking, and reopening
- Added jail intake report generation
- Migrated jail intake claims and roles
- Renamed jail tables to plural and added jail code descriptions

## Jail Observation

- Added the initial Jail Observation UI
- Stubbed in observation passes and a refactored ObservationPassService
- Added pass-grid validation with redirect on invalid input

## OCA Reporting

- Added OCA Court Identifier to agencies
- Added handling for negative amounts and refactored category mapping in the OCA XML generator
- Added compliance-dismissal handling and OCA disposition guardrails
- Added Total Disposed calculation
- Added Juvenile OCA support, including age handling
- Added handling for "other activities" and fines collected in OCA reporting

## Payment & Citation

- Added citation payment eligibility checks and email receipts
- Added the online payment URL to court violation reports
- Added a payment clean-up utility and a payment reallocation tool
- Added Stripe payout batch creation and handling
- Added the ability to print citations from a court violation

## Architecture Changes

- Added a court violation backfill history script
- Added new pipeline configuration files (deployment and vNext)
- Added database name configuration parameterized by agency name and environment
- Added Demo 2 configuration
- Updated client-side libraries (including QuestPDF 2025.12.4)
- Updated server-side libraries
- Made other small improvements and fixes
