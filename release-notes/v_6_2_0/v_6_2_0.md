## General Changes

- Refactored toast notifications and API error handling for clearer error messages
- Updated icons and small Vuetify plugin updates
- Made other small improvements and fixes

## Jail Management (JLM)

- Introduced the Jail Management module, a real-time command center for in-custody operations
- Added core JLM entities and service layer for jail management
- Added task scheduling, instance generation, and SignalR notifications
- Added the Observation Pass Service with cell check activity codes, idempotent entry handling, and group-scoped episode handling
- Added the Pass Runner with cell check configuration, pass handling, and overdue grace periods
- Added Medication Management with administration and scheduling, task schedules, and a work queue panel
- Added a Visitor Log (behind a feature flag)
- Added Booking Health checks and validation API enhancements
- Added a Live Feed for real-time updates and presence management for read-only bookings
- Added facility location management with location validation, type configuration, and drag-and-drop roster management
- Added active episode occupancy count, episode filtering, and drag-and-drop episode handling
- Added a booking checklist rail
- Added default Texas jail schedules
- Added current-pass tracking on the heartbeat bar and board stats
- Added a JLM command-center stylesheet and updated styling on cards, episode chips, and pass runner

## Jail Intake

- Added Mental Health Screening fields and report generation in the jail intake process
- Added a Document Vault service for storing booking-related files
- Added driver license scanning to jail intake entry
- Added property bag updates and person step completion
- Added direct step status control (replacing the prior "override step" functionality)
- Added an API to retrieve bookings by custody episode
- Added arrestee type determination based on date of birth in the custody step
- Added jail intake bookings to the record preview
- Updated booking-step components, header layout, and validation styling
- Disabled the DOCS step pending further work
- Migrated jail intake to use master person snapshots for stable historical data
- Added "Include In Cell Checks" code types

## Master Person Duplicate Detection

- Added master person duplicate detection with API endpoints and data models
- Added cluster management services for duplicates
- Updated AdminMasterMerge with cluster detail retrieval, table display, cluster filtering, and support for new duplicate entities (location, organization, property, vehicle)

## Master Records & Audit

- Added a two-phase save flow for master entities
- Refactored audit entry processing and audit handling

## Mobile Citation

- Added debounced search in the address quick-control
- Refactored Google Maps placeAutocomplete for compatibility and performance
- Improved loading-state management and PDF generation (print scaling, dynamic height, dimension handling)
- Replaced the legacy debounce dependency with VueUse's useDebounceFn

## Architecture Changes

- Refactored SQL migration scripts for idempotency
- Added new pipeline configurations: solution file, NuGet restore, optimized test command
- Added Azure Government compatibility for Directory API URLs
- Added Bicep and database deployment updates with long-term retention policy
- Added tenant-specific Descope configuration sourced from the Directory API
- Added new validation, autocomplete, and stepper components in the UI
- Added new search stores for medication log, observation pass, and visitor log
- Added jail-intake and JLM routes
- Added documentation for jail requirements, jail intake, and a medication diagnostic SQL script
- Updated client-side libraries
- Updated server-side libraries
- Made other small improvements and fixes
